# 30-01 Генерация плана тренировок (LLM) + сохранение в структуру Plan→Exercise

## Business goal
После онбординга пользователь получает персональный план. LLM нужен в основном на старте, но результат должен быть нормализован: план содержит ссылки на Exercise.

## Scope
- Оркестратор генерации:
  1) взять `UserProfile`
  2) вызвать LLM и получить план (дни → упражнения)
  3) для каждого упражнения: матчинг в каталоге (поиск), иначе создать `Exercise`
  4) создать `TrainingPlan` + `TrainingDay` + `PlanExercise(exerciseId, orderIndex)`
- Хранить 1 активный план (MVP): новый план делает старый неактивным.

## UX flow
- После завершения анкеты:
  - «Сгенерировать план»
  - затем «Показать план»
- В меню: «Планы тренировок» → «Активный план»

## Acceptance criteria
- План генерируется и сохраняется в БД.
- `PlanExercise` ссылается на `Exercise.id`.
- Повторная генерация не создает дубликаты упражнений при совпадении.

## Technical notes
- В MVP допустим простой матчинг по имени + нормализация (trim/lower).
- Ошибки LLM: ретрай 1–2 раза, иначе понятное сообщение пользователю.

## Smoke test
- Новый пользователь → анкета → генерация → просмотр плана.
