# 30-02 Просмотр и корректировка плана

## Business goal
Пользователь должен видеть план заранее и иметь возможность корректировать его (минимально) без повторной анкеты.

## Scope (MVP)
- Просмотр активного плана по дням.
- Корректировки:
  - заменить упражнение в дне (выбор из каталога через поиск/фильтры)
  - изменить порядок упражнений (up/down)
  - добавить/удалить упражнение (опционально, но желательно)

## UX flow
- «Планы тренировок» → выбрать день → список упражнений
- На упражнении: «Заменить», «Вверх», «Вниз», «Удалить»
- «Добавить упражнение» → поиск по названию

## Acceptance criteria
- Изменения плана сохраняются и отражаются в тренировке.
- Удаление/замена не ломает историю прошлых тренировок.

## Technical notes
- Переупорядочивание через `orderIndex`.

## Tests
- Замена/добавление/удаление, корректность orderIndex.
