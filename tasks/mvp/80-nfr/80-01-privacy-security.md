# 80-01 Privacy & Security (MVP)

## Business goal
Гарантировать изоляцию данных по пользователям и базовую безопасность.

## Scope
- Все запросы к данным scoped по `telegram_user_id`.
- Команды админа (если есть) — отдельный allowlist user_id.
- Логи без персональных данных (не логировать вес/фото; максимум IDs).

## Acceptance criteria
- Невозможно получить данные другого пользователя через бота.
- В репозитории нет токенов/секретов.

## Tests
- Негативные сценарии (подмена user_id невозможна через Telegram).
